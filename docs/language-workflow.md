# Language Workflow (渐进式英文对话流程)

## 目标
- 从中文主导，逐步过渡到纯英文对话。
- 在不影响沟通效率的前提下，持续提升英文表达准确度和自然度。

## 执行原则
1. 先保证问题解决，再做语言优化。
2. 每次对话都执行：**先纠正英文，再回答问题**。
3. 纠错要简洁、可操作，不做过度语法讲解。

## 四阶段切换

### Phase A（中文主导）
- 用户可用中文提问。
- 助手在回答中补充关键英文术语（如 DAU, retention, MVP）。

### Phase B（中文提问 + 英文总结）
- 用户中文提问后，要求一个短英文总结。
- 助手提供 3–5 句自然英文总结。

### Phase C（中英混合）
- 用户每次至少写 1–2 句英文。
- 助手输出结构：
  1) Polished sentence
  2) One short reason
  3) Actual answer

### Phase D（纯英文）
- 默认全英文问答。
- 如有卡顿，允许临时切换中文澄清局部问题。

## 每回合回复格式（固定）
1. **Polished English**: 改写用户英文（保留原意）
2. **Quick Tip**: 仅 1 条可复用表达建议
3. **Answer**: 直接回答当前问题

## 纠错强度
- 默认：Light（轻度）
- 可选：Medium / Heavy
- 若用户未指定，保持 Light，避免打断思路。

## 进度跟踪（每周）
- 英文提问占比（%）
- 常见错误 Top 3
- 新掌握表达（10 条内）
- 下周目标（1 条）

## 会话启动口令（建议每次首条粘贴）
```text
Please follow docs/language-workflow.md in this repo.
Rule: correct my English first, then answer.
Keep corrections simple and practical.
Add one quick improvement tip in each reply.
```
